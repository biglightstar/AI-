## 讲解内容

### 1. 为什么需要版本控制

学生反馈遇到过"改坏了回不去、桌面一堆最终版文件"的痛点 → 引出 Git 的存档点概念：一份文件 + 后台自动记录所有版本，随时可回到任意版本。

### 2. Git 的"时间线"模型

- 存档点 = commit，每个 commit 有编号
- 回到旧版本修改会形成分支（branch），不覆盖后续版本
- 用"论文初稿 → 写完全文 → 导师批注版 → 最终版"的比喻讲解

### 3. 命令行实操（第一次接触）

| 命令 | 含义 | 学生操作情况 |
|------|------|:--:|
| `git --version` | 查看版本 | ✅ 2.54.0 |
| `git config user.name` | 查看用户名 | ✅ biglightstar |
| `git config user.email` | 查看邮箱 | ✅ 3053984178@qq.com |
| `git init` | 初始化仓库 | ✅ 在 AI学习 目录下 |
| `git status` | 查看仓库状态 | ✅ 理解了 untracked/staged 的区别 |
| `git add .` | 添加文件到暂存区 | ✅ 出现 LF/CRLF 警告（已解释） |
| `git commit -m "..."` | 提交存档 | ✅ 第一步存档完成 |

### 4. 遇到的小问题

| 问题 | 解决 |
|------|------|
| 路径写错（`/g/...` vs `G:\...`） | 区分了 Git Bash 和 cmd 的路径写法 |
| 在 C 盘误建了仓库 | `rmdir /s .git` 删除 |
| LF/CRLF 警告 | 解释了 Windows vs Linux 换行符差异 |
| 学生担心记不住命令 | 讲解了英文原词（init=initialize, status=状态, add=添加, commit=提交, -m=message） |

### 5. 核心口诀

> **改 → add → commit**（三步循环）
