## 讲解内容

### 1. GIT1.2 Git 安装与配置

- 讲解 `git config` 的三级配置（system/global/local）
- 学生亲手打开 `.gitconfig` 文件查看内容
- 理解了 `git config --list --show-origin` 输出
- **学生反馈**：觉得配置话题太慢，要求加快节奏

### 2. GIT1.3 Git 工作流程 + GIT1.4 三区模型深入

- 通过 `echo > file.txt` 创建文件 → `git add` → `git commit` 完整实验
- 详解工作区/暂存区/仓库的三区关系
- 重点讲解 `git add` 双重作用：①开始追踪新文件 ②更新暂存区
- 学生发现了 LF/CRLF 警告，已解释为正常现象
- **学生核心理解**：`git add` = 工作区→暂存区，`git commit` = 暂存区→仓库

### 3. `git diff` 深入

- `git diff`：工作区 ↔ 暂存区
- `git diff --staged`：暂存区 ↔ 最新 commit（学生实操为空）
- `git diff HEAD`：工作区 ↔ 最新 commit
- `git diff <commitA> <commitB>`：两次提交对比（学生成功查看差异）
- 解释了 `-` 和 `+` 的含义

### 4. `git log` 深入

- `git log`（完整版）、`git log -2`（限制条数）
- `git log -p`（看具体改动）vs `git log --stat`（看文件统计）
- `git log --author` 按作者过滤
- `git log --since` 按时间过滤（发现 `2026-07-14` 无输出——时区问题）
- `git log --oneline --graph --all`（预习分支图形）

### 5. `git commit -a` 范围验证实验

- 学生实验：修改已跟踪文件 + 创建新文件 → `git commit -a`
- **结果**：已跟踪文件被提交 ✅，新文件留在 Untracked ❌
- **学生核心理解**：`git commit -a` 只对已跟踪文件生效，新文件必须手动 `git add`

### 6. `git reset` 撤销操作

- `git reset <文件>`：暂存区→工作区（学生比喻为"add 的减法"）
- `git reset HEAD~1`：撤销最近一次 commit，保留修改在工作区
- 学生成功完成撤销-恢复流程

### 7. `git restore` 新版撤销命令

- `git restore <文件>`：丢弃工作区修改，恢复成暂存区版本（危险，内容丢失）
- `git restore --staged <文件>`：从暂存区撤回，内容不丢
- 成功实验后学生理解了两者的区别
- **连接英文原意**：restore = 恢复、还原

### 8. `git add` 变体

- `git add .` vs `git add -A` vs `git add <文件>`
- `-A` = `--all` = 整个仓库

- **话题**：GIT2.1 init/clone 概念 + GIT2.3 blame + GIT2.4 tag + GIT2.5 stash

### 讲解内容

1. **仓库根目录与路径**
   - 学生不理解"仓库根目录"概念，通过 `.git` 位置推导根目录
   - 理解了 `AI学习/AI学习/` 是子文件夹而非根目录
   - 理解了 `git blame progress/Git-study.md` 的路径是从当前目录出发
   - 用"地址"比喻解释文件路径层级

2. **GIT2.4 `git tag` 标签管理**
   - 理解 tag = 给 commit 打"有意义的名字"
   - 实操创建 `v1.0.0` 标签
   - 出现了 `git tag -1`（数字1）错误——输出了帮助文档
   - 修正为 `git tag -l`（字母l）正确显示标签列表
   - `git log --oneline --decorate` 看到 `tag: v1.0.0`

3. **GIT2.3 `git blame` 逐行追责**
   - 学生实操 `git blame progress/Git-study.md`
   - 理解了每一行格式：commit哈希/作者/日期+时区/行号
   - 发现大部分行来自第一次 commit（`c4d0f33`）

4. **GIT2.5 `git stash` 暂存工作现场**
   - 学生实操 stash → stash list → stash show
   - 理解 stash = "把半成品藏起来"→ pop 取回
   - 学生提出「为什么 `stash list` 看不到文件名」→ `stash show` 解决
