## 讲解的概念

### 1. 短路求值与边界检查顺序（核心纠错）

**问题代码**：
```cpp
while(nums[i] != 0 && i<n)  // ❌ 先访问再检查边界
```

**讲解要点**：
- C++ 的 `&&` 是短路求值，从左到右执行
- 当 `i == n` 时，`nums[i]` 已经越界，但程序先执行 `nums[i] != 0` 才检查 `i < n`
- 正确写法：`while(i < n && nums[i] != 0)` — 边界检查永远放左边

### 2. 空间复杂度分析

- 只用了 4 个 int 变量（n, i, maxcount, count）
- 输入数组是引用传递，不产生拷贝
- 结论：O(1) 常数级，最优解

### 3. LeetCode 485 完整总结

- 两种标准写法：单指针 + 内层 while vs 单层 for 循环
- 核心坑点表格：短路求值顺序、结尾是1的情况、空数组
- 复杂度结论：时间 O(n)，空间 O(1)
- 延伸：487（翻转 1 个 0）、1004（翻转 K 个 0）— 升级到真正的滑动窗口
