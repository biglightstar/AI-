## G1. Go 基础入门

### G1.1 Go 语言概述与设计哲学

**诞生背景**：三位创始人（Thompson/Pike/Griesemer）因 C++ 编译太慢而创造 Go

**9 大设计哲学**：

| 序号 | 设计哲学 | 说明 |
|:---:|---------|------|
| 1 | **编译快** | 无头文件，所有依赖自动解析 |
| 2 | **25 关键字** | 极简语法，比 C（37）+ C++（84）少得多 |
| 3 | **大小写导出** | 大写字母开头 = 公开（public），小写 = 私有（private） |
| 4 | **错误是值** | 错误通过返回值处理，不是异常机制 |
| 5 | **struct + 组合** | 组合优于继承，没有类（class） |
| 6 | **goroutine** | 轻量级协程，go 关键字启动 |
| 7 | **channel** | 通过通信共享内存，不是共享内存来通信 |
| 8 | **自动 GC** | 垃圾回收，无需手动管理内存 |
| 9 | **零值安全** | 变量声明后自动有默认值，不会出现未初始化 |

**核心理解——"通过通信共享内存"**：
- 传统方式（C++ mutex）：多线程共享一块内存，用锁保护
  ```cpp
  std::lock_guard<std::mutex> lock(mtx);
  shared_data = new_value;
  ```
- Go 方式（channel）：谁需要数据，就创建一条通道把数据发过去
  ```go
  ch <- data  // 发送
  result := <-ch  // 接收
  ```

### G1.2 环境搭建与工具链

**三大环境变量**：

| 变量 | 作用 |
|------|------|
| **GOROOT** | Go 安装目录（`F:\Go`） |
| **GOPATH** | 工作区目录（`C:\Users\admin\go`），存放下载的第三方包 |
| **GOMODCACHE** | 模块缓存目录，存放已下载的模块 |

**Go 工具链命令**：

| 命令 | 英文原意 | 功能 |
|------|---------|------|
| `go run main.go` | run（运行） | 直接运行代码，不生成可执行文件 |
| `go build -o 名称.exe .` | build（构建） | 编译成 .exe 可执行文件 |
| `go mod init 模块名` | module + initialize | 初始化模块，生成 go.mod |
| `go mod tidy` | tidy（整理） | 清理无用依赖，添加缺失依赖 |
| `go fmt` | format（格式化） | 自动格式化代码（统一风格） |

**go run vs go build**：
- `go run`：调试阶段用，想看能不能运行就用它
- `go build`：发布阶段用，生成 .exe 可执行文件

**Go 为什么不需要头文件**？
- 头文件编译花费大量时间
- Go 通过 `go.mod` + 自动依赖解析替代头文件

**多包项目结构**：
```
项目目录/
├── go.mod          # 模块身份证
├── main.go         # 入口文件（package main）
└── helper/
    └── greet.go    # 辅助包
```

- `main` 包是特殊包，编译成可执行文件
- 其他包通过 `import "模块名/包路径"` 导入
- Go 自动解析依赖，无需手动配置

## Go 学习原则

1. **不只学语法，要学设计哲学**——理解 Go 为什么这样取舍
2. **不只写 Demo，要做对比**——每个概念和 C++/Python/JS 对比
3. **不只背规则，要写代码验证**——Go 编译器的严格性是最好的老师
